  1. Sistema operativo: Windows
  2. Software instalado:
    • BaseX
    • PHP
    • Servidor web (XAMPP, WAMP o similar)
    • Archivo XML de entrada (ej. eventos.xml)

✅ 0. CONFIGURAR PHP

Utilitzar un servidor PHP:

  • Configurar php.ini descomentando la linea: extension=php_sockets.dll
  • Crear un nou projecte PHP dins la carpeta (httpdocs) de Xamp.
  • Iniciar el servidor PHP, i comprovar que funciona.

✅ 1. CONFIGURAR BASEX COMO SERVIDOR (ACTUALIZADO)

1.1. Iniciar el servidor HTTP de BaseX

  1. Abre la carpeta donde instalaste BaseX.
  2. Ejecuta basexhttp.bat (doble clic o desde consola).
    • Esto inicia BaseX HTTP Server en:
      http://localhost:8984.
    • Usuario y contraseña por defecto:
      admin / admin

1.2. Cambiar contraseña del usuario admin

Tienes dos formas: desde consola o desde la GUI.

🔧 Opción A: Desde consola

Abre basexclient.bat (cliente en modo consola) y ejecuta:

PASSWORD admin nueva_contrasena

Esto cambia la contraseña del usuario admin a nueva_contrasena.

🖥 Opción B: Desde la GUI (BaseX.exe)

  1. Abre BaseX.exe
  2. Menú: Database > Manage Users
  3. Selecciona admin y haz clic en Edit Password

1.3. Crear un nuevo usuario

🔧 Opción A: Desde consola (cmd) 

CREATE USER usuario_nuevo contraseña123
GRANT READWRITE ON mi_bd TO usuario_nuevo

  • También puedes dar acceso global con:

    GRANT ADMIN TO usuario_nuevo

🖥 Opción B: Desde la GUI

  1. Abre BaseX.exe
  2. Menú: Database > Manage Users
  3. Clic en Create, rellena:
    • Username
    • Password
    • Permisos (READWRITE, ADMIN, etc.)
    • Base de datos

1.4. Usar nuevo usuario desde PHP

Si creaste por ejemplo:

usuario: pepe
contraseña: clave123

Entonces en el script PHP:

$session = new Session("localhost", 1984, "pepe", "clave123");

2. Importar el archivo XML como base de datos

Puedes usar la interfaz gráfica de BaseX o hacerlo por consola.

Opción A: Usar GUI de BaseX

  1. Abre BaseX.exe.
  2. Ve a Database > New.
  3. Ponle un nombre (ej. mi_bd).
  4. Selecciona el archivo XML (datos.xml) y haz clic en Create.

Opción B: Desde terminal (cmd) 

basexclient -U admin -P admin -c "CREATE DB mi_bd C:\ruta\a\datos.xml"

3. Preparar PHP para conectarse a BaseX

BaseX usa el protocolo Socket para conectarse. En el repositorio oficial hay una clase llamada Session.php que facilita esto.

⚙️ 3.1 Descargar la clase Session.php

Descarga desde:
https://github.com/BaseXdb/basex/blob/master/basex-api/src/main/php/Session.php

Guárdala en el mismo directorio que tu script PHP.

4. Crear un script PHP de prueba 

<?php
require_once("Session.php");

try {
    // Conectar con el servidor BaseX
    $session = new Session("localhost", 1984, "admin", "admin");

    // Ejecutar una consulta XQuery
    $result = $session->execute("XQUERY db:open('mi_bd')");

    echo "<pre>Resultado:\n" . htmlspecialchars($result) . "</pre>";

    // Cerrar la sesión
    $session->close();
} catch (Exception $e) {
    echo "Error: " . $e->getMessage();
}
?>

⚠️ Asegúrate de que el puerto 1984 (por defecto para BaseX) esté abierto.

5. Modificar el XML desde PHP

Puedes enviar comandos XQuery desde PHP para modificar el XML:

<?php
require_once("Session.php");

try {
    $session = new Session("localhost", 1984, "admin", "admin");

    // Define el nuevo evento como nodo XML
    $nuevoEvento = <<<XQUERY
XQUERY
  insert node
    <evento>
      <id>3</id>
      <nombre>Taller</nombre>
      <fecha>2025-04-15</fecha>
      <descripcion>Taller de robótica educativa</descripcion>
    </evento>
  into db:open('eventos_db')/eventos
XQUERY;

    // Ejecutar la consulta
    $session->execute($nuevoEvento);

    echo "✅ Evento insertado correctamente.";

    $session->close();
} catch (Exception $e) {
    echo "❌ Error: " . $e->getMessage();
}
?>

<?php
/*
 * This example shows how database commands can be executed.
 *
 * Documentation: https://docs.basex.org/wiki/Clients
 *
 * (C) BaseX Team, BSD License
 */
include_once 'load.php';

use BaseXClient\BaseXException;
use BaseXClient\Session;

try {
    // initialize timer
    $start = microtime(true);
    // create session
    $session = new Session("localhost", 1984, "admin", "admin");

    // perform command and print returned string
    print $session->execute("xquery 1 to 10");

    // close session
    $session->close();

    // print time needed
    $time = (microtime(true) - $start) * 1000;
    print "\n$time ms\n";
} catch (BaseXException $e) {
    // print exception

    print $e->getMessage()&"abC";

}

Verificar que el usuario admin tiene acceso a BaseX de varias formas

✅ OPCIÓN 1: Usar el cliente en consola (basexclient.bat)

  1. Abre una terminal (cmd o PowerShell).
  2. Ve a la carpeta donde instalaste BaseX.
  3. Ejecuta:
basexclient.bat -U admin -P admin

Si ves algo como esto:

> BaseX Client [Server]
>
> >

✅ El usuario admin tiene acceso.

Si falla, verás un error tipo:

Access denied.

✅ OPCIÓN 2: Usar la interfaz web (http://localhost:8984)

  1. Abre tu navegador y entra en:
    http://localhost:8984
  2. Ingresa el usuario admin y su contraseña.
  3. Si accedes al panel web (REST interface), está funcionando.

✅ Si aparece una pantalla de login repetidamente o un error 401, las credenciales son incorrectas o el servidor no está corriendo.

✅ OPCIÓN 3: Probar con PHP

Crea un pequeño script test.php en tu servidor local:

<?php
require_once("Session.php");

try {
    $session = new Session("localhost", 1984, "admin", "admin");
    echo "✅ Conexión exitosa con el usuario admin.";
    $session->close();
} catch (Exception $e) {
    echo "❌ Error de conexión: " . $e->getMessage();
}
?>
  1. Colócalo en htdocs (XAMPP) o el directorio raíz de tu servidor.
  2. Visita http://localhost/test.php.

❗ ¿Qué hacer si falla?

  • Verifica que el servidor BaseX esté en ejecución (basexhttp.bat y/o basexserver.bat).
  • Asegúrate de que el puerto 1984 no esté bloqueado por firewall.
  • Si cambiaste la contraseña de admin, actualízala en el script o consola.
  • Puedes reiniciar la configuración de usuarios borrando .\data\etc\users.xml (⚠️ esto resetea usuarios y contraseñas).
